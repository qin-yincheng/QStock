## 一、缠论核心三大理念

这是整个缠论的灵魂，不理解这三点，就无法真正用好缠论。

### 1.1 不测而测

> "技术分析的最终意义不是去预测市场要干什么，而是市场正在干什么，是一种当下的直观。在市场上所有的错误都是离开了这当下的直观，用想象、用情绪来代替。"

**核心含义**：

- **不要预测**未来的顶部/底部在哪里
- **只关注**市场当下正在发生什么结构
- 技术分析是"读走势"，不是"猜走势"
- 根据走势本身的规律（分型→笔→线段→中枢）来决策

**常见错误（反面教材）**：

| 错误思维 | 正确思维 |
|---------|---------|
| "我觉得还能涨，再等等" | "当前结构是什么？买卖点是否出现？" |
| "底部应该在 XX 元附近" | "底分型是否形成？笔是否完成？" |
| "大盘要跌了" | "当前中枢方向如何？有无背驰信号？" |

---

### 1.2 完全分类

> "对于一个真正的操作者，没有任何情况是意外的，因为，所有的情况都被完全地分类了，所有相应的对策都事先有了，只是等着市场自己去选择，去触及我们事先给定的开关。"

**核心含义**：

- 事先把所有可能发生的走势**穷举分类**
- 为每一种情况准备好**对应策略**（开关）
- 不依赖预测，只等市场触发开关

**实战模板——完全分类示例**：

```
当前持仓中，股价处于中枢内震荡：
  → 情况A：向上突破中枢 ZG → 三买信号，加仓或持有
  → 情况B：向下跌破中枢 ZD → 止损离场
  → 情况C：继续在中枢内震荡 → 等待，不操作
```

---

### 1.3 级别思维

> "学了缠论的理论，脑子里必须时刻有两个字：级别。如果连级别都搞不清楚，你还419？被419还差不多。"

**核心含义**：

- 任何分析都必须**明确级别**
- 不同级别有完全不同的操作策略和持仓时间
- **混淆级别是缠论实操中最常见、最致命的错误**

**级别对照表**：

| 级别 | K线周期 | 持仓时间 | 适合人群 |
|------|---------|---------|--------|
| 1分钟级别 | 1分钟K线 | 分钟~小时 | 高频交易者 |
| 5分钟级别 | 5分钟K线 | 小时~半天 | 日内交易者 |
| 30分钟级别 | 30分钟K线 | 1~3天 | 短线交易者 |
| **60分钟级别** | **60分钟K线** | **3~10天** | **本项目目标** |
| 日线级别 | 日线K线 | 1~4周 | 中线交易者 |
| 周线级别 | 周K线 | 1~3月 | 波段交易者 |
| 月线级别 | 月K线 | 数月~数年 | 长线投资者 |

> **本项目**：基于60分钟K线做缠论分析，目标持仓3~10天，属于短中线策略。

## 二、基本构成要素：分型（FX）

分型是缠论的**最小识别单元**，是笔的起点和终点。

### 2.1 K线包含关系处理（必做预处理）

在识别分型之前，**必须先处理K线的包含关系**，这是缠论的基础规则。

**包含关系定义**：
若某根K线的高点和低点都被相邻K线"包住"（或"包住"相邻K线），则称为包含关系。

```
包含条件（满足任意一个）：
  high[i] >= high[i+1] AND low[i] <= low[i+1]  （i 包含 i+1）
  high[i] <= high[i+1] AND low[i] >= low[i+1]  （i+1 包含 i）
```

**处理规则**：

| 所处位置 | 处理方式 |
|---------|--------|
| 上升趋势中 | 取两根K线的**最高高点**和**最高低点**（向上合并）|
| 下降趋势中 | 取两根K线的**最低高点**和**最低低点**（向下合并）|

处理后的K线序列称为**独立K线序列**，才能用于分型识别。

---

### 2.2 顶分型（Mark.D）

> 注意：czsc 库中顶分型用 `Mark.D`（D = 顶），底分型用 `Mark.G`（G = 底）。

**顶分型定义**：连续三根独立K线，中间那根的高点最高、低点也最高。

```
顶分型判断：
  K[i-1].high < K[i].high > K[i+1].high
  K[i-1].low  < K[i].low  > K[i+1].low
  → 中间 K[i] 构成顶分型
```

**图示**：
```
         ___
        |   |
   ___  |   |  ___
  |   | |   | |   |
  |   | |   | |   |
  K[-1]  K[0]  K[1]
         ↑
       顶分型
```

**czsc 中的顶分型**：
```python
from czsc.objects import Mark

fx = c.fx_list[-1]
if fx.mark == Mark.D:   # 顶分型
    print(f"顶分型：时间={fx.dt}, 最高价={fx.high}, 分型价={fx.fx}")
```

---

### 2.3 底分型（Mark.G）

**底分型定义**：连续三根独立K线，中间那根的高点最低、低点也最低。

```
底分型判断：
  K[i-1].high > K[i].high < K[i+1].high
  K[i-1].low  > K[i].low  < K[i+1].low
  → 中间 K[i] 构成底分型
```

**czsc 中的底分型**：
```python
if fx.mark == Mark.G:   # 底分型
    print(f"底分型：时间={fx.dt}, 最低价={fx.low}, 分型价={fx.fx}")
```

---

### 2.4 分型的关键属性（czsc）

```python
fx = c.fx_list[-1]

fx.dt          # 分型时间（中间K线的时间）
fx.mark        # Mark.D（顶分型）或 Mark.G（底分型）
fx.fx          # 分型价格：顶分型=中间K线最高价，底分型=中间K线最低价
fx.high        # 三根K线中的最高价
fx.low         # 三根K线中的最低价
```

> **注意**：`fx.fx` 是分型的**特征价格**，顶分型时等于最高价，底分型时等于最低价。

---

### 2.5 分型的有效性条件

两个相邻分型（一顶一底）必须满足**至少有一根独立K线不重叠**，才认为是有效分型对。

```
有效分型对条件：
  顶分型与底分型之间，至少间隔1根独立K线
  （即两个分型不能紧紧相邻，中间至少有1根K线"隔离"）
```

## 三、基本构成要素：笔（BI）

笔是由相邻一顶一底（或一底一顶）分型连接而成的价格运动单元。

### 3.1 笔的构成条件

**上升笔**：底分型 → 顶分型
- 起点（底分型）的最低价 < 终点（顶分型）的最高价
- 两个分型之间**至少有5根独立K线**（含两端分型的中间K线，共5根）
- 实际上等价于：两个分型之间至少有1根不属于任何分型的独立K线

**下降笔**：顶分型 → 底分型
- 起点（顶分型）的最高价 > 终点（底分型）的最低价
- 同样需要至少5根独立K线

```
最简上升笔示意（5根独立K线）：

                    ___
                   |   |
         ___       |   |
        |   |  K3  |   |
   ___  |   |      |   |
  |   | |   |      |   |
  |   |                 
  底分型      间隔K线   顶分型
```

---

### 3.2 笔的关键属性（czsc）

```python
from czsc.objects import Direction

bi = c.bi_list[-1]

bi.direction   # Direction.Up（上升笔）或 Direction.Down（下降笔）
bi.high        # 笔的最高价
bi.low         # 笔的最低价
bi.fx_a        # 起点分型（FX 对象）
bi.fx_b        # 终点分型（FX 对象）
bi.fx_a.dt     # 笔起点时间
bi.fx_b.dt     # 笔终点时间

# 判断方向
if bi.direction == Direction.Up:
    print(f"上升笔：{bi.fx_a.dt} ~ {bi.fx_b.dt}, {bi.low:.2f} → {bi.high:.2f}")
if bi.direction == Direction.Down:
    print(f"下降笔：{bi.fx_a.dt} ~ {bi.fx_b.dt}, {bi.high:.2f} → {bi.low:.2f}")
```

---

### 3.3 笔的序列特性

- 笔总是**交替方向**：上升笔 → 下降笔 → 上升笔 → …
- `c.bi_list[-1]` 是**最新的已完成笔**（或当前未完成笔）
- `c.bars_ubi` 是**当前未完成笔**的K线列表

```python
# 判断当前未完成笔方向
bars_ubi = c.bars_ubi
if len(bars_ubi) > 0:
    trend = "上升" if bars_ubi[-1].close > bars_ubi[0].open else "下降"
    print(f"当前未完成笔方向：{trend}")
```

---

### 3.4 实战意义

| 笔的状态 | 含义 | 操作参考 |
|--------|------|--------|
| 下降笔结束（底分型成立）| 下跌动能可能衰竭 | 关注买点 |
| 上升笔结束（顶分型成立）| 上涨动能可能衰竭 | 关注卖点 |
| 笔尚未完成（bars_ubi）| 趋势仍在延续 | 持有/等待 |

## 四、线段

### 4.1 线段的定义

线段由**至少3笔**构成，是比笔更高一级的价格运动单元。

**上升线段**：以下降笔开始，整体方向向上，由奇数笔（3、5、7…）组成。
**下降线段**：以上升笔开始，整体方向向下，由奇数笔（3、5、7…）组成。

**线段成立条件**（简化版）：
```
三笔构成上升线段的条件：
  笔1（下降）+ 笔2（上升）+ 笔3（下降）
  → 笔3的低点 > 笔1的低点（不创新低）
  → 笔2的高点 > 笔0（前一个上升笔）的高点（创新高）
  即：整体高点和低点都在抬升
```

---

### 4.2 线段与笔的关系

```
线段级别 > 笔级别

一个线段内部包含多笔：
上升线段示意：
  ↗（笔1上升）↘（笔2下降）↗（笔3上升）↘（笔4下降）↗（笔5上升）...
  但整体高低点持续抬升 → 上升线段
```

> **czsc 库**：`c.bi_list` 中的笔是基础元素，线段识别需通过更高级的 `BI` 对象或手动实现。

## 五、中枢（Zhongshu）

中枢是缠论最重要的概念之一，理解中枢是理解三类买卖点的基础。

### 5.1 中枢的定义

**中枢**：至少三笔的价格重叠区间。

```
设三笔为：笔A、笔B、笔C
ZG（中枢高点）= min(笔A.high, 笔B.high, 笔C.high)  # 三笔最高点中的最低
ZD（中枢低点）= max(笔A.low, 笔B.low, 笔C.low)   # 三笔最低点中的最高

中枢成立条件：ZG > ZD（存在重叠区间）
```

**图示**：

```
价格轴
  ↑
  |         ___________
  |        |           |  ← ZG（中枢高点）
  |   _____|_____      |
  |  |     |     |    |
  |  |     |_____|    |  ← ZD（中枢低点）
  |  |_____|
  |  笔A(↑) 笔B(↓)  笔C(↑)
  +------------------------→ 时间
       ←── 中枢 ──→
       [ZD, ZG] 重叠区间
```

---

### 5.2 中枢的扩展

当走势在中枢内反复震荡，超过3笔时，中枢可以**扩展**：

```
中枢扩展条件：
  新进入中枢的笔（笔D、笔E...）与现有中枢仍有重叠
  → 中枢持续，更新 ZG/ZD

中枢结束条件（出现以下任意一种）：
  → 一笔完全突破 ZG（向上离开中枢）→ 可能是三买
  → 一笔完全跌破 ZD（向下离开中枢）→ 可能是三卖
```

---

### 5.3 中枢的级别

中枢的级别由构成它的**笔的级别**决定：

| 构成元素 | 中枢级别 |
|---------|--------|
| 60分钟笔构成 | 60分钟中枢 |
| 日线笔构成 | 日线中枢 |
| 周线笔构成 | 周线中枢 |

**高级别中枢 = 多个低级别中枢的组合**

---

### 5.4 czsc 中枢识别（手动实现）

czsc 库的 `bi_list` 提供原材料，中枢识别需手动计算：

```python
def find_zhongshu(bi_list):
    """从笔列表中识别最近的中枢，返回 (ZG, ZD) 或 None"""
    if len(bi_list) < 3:
        return None
    # 从最近的笔开始往前找
    for i in range(len(bi_list) - 2, 1, -1):
        three = bi_list[i-2:i+1]
        zg = min(bi.high for bi in three)  # 三笔最高点中的最低
        zd = max(bi.low for bi in three)   # 三笔最低点中的最高
        if zg > zd:
            return (zg, zd)
    return None

# 使用
zs = find_zhongshu(c.bi_list)
if zs:
    zg, zd = zs
    print(f"最近中枢：ZG={zg:.2f}, ZD={zd:.2f}")
```

## 六、三类买卖点

三类买卖点是缠论操作的**核心信号系统**，建立在中枢理论之上。

### 6.1 三类买点总览

| 类型 | 触发条件 | 风险 | 收益 | 操作激进程度 |
|-----|---------|------|------|------------|
| **一买**（第一类买点）| 下跌趋势末端，最后一笔下降笔出现背驰 | 最高 | 最大 | 最激进 |
| **二买**（第二类买点）| 一买后反弹，回调不破一买低点 | 中等 | 较大 | 适中 |
| **三买**（第三类买点）| 上升笔突破中枢后，回调不破中枢 ZG | 最低 | 较小 | 最保守 |

---

### 6.2 第一类买点（一买）

**定义**：在下跌趋势中，最后一段下降笔与前一段同级别下降走势相比出现**背驰**，此时形成的底分型即为一买。

**触发条件**：
1. 存在明显下降趋势（多笔下降走势）
2. 最后一笔下降笔出现背驰（MACD 面积缩小）
3. 底分型形成确认

**操作逻辑**：
```
一买信号出现 → 轻仓试探性买入
止损：跌破一买底分型最低价
目标：等待二买确认后加仓
```

**风险提示**：一买是**风险最高**的买点，因为可能出现"假底"继续下跌。建议仓位不超过 20-30%。

---

### 6.3 第二类买点（二买）

**定义**：一买之后，价格反弹形成一定走势（上升笔或中枢），随后回调，**回调低点不破一买低点**，此时形成的底分型为二买。

**触发条件**：
1. 一买已经形成
2. 价格从一买反弹（至少形成一笔上升笔）
3. 回调低点 > 一买最低价（不创新低）
4. 回调形成底分型

**操作逻辑**：
```
二买信号出现 → 中等仓位买入（可结合一买合计达到半仓）
止损：跌破一买低点
目标：等待突破前期高点
```

**与双龙战法结合**：二买区域叠加 EMA5/EMA10 金叉 + MACD 金叉，即为高确定性入场点。

---

### 6.4 第三类买点（三买）

**定义**：上升笔突破中枢（价格 > ZG），随后回调，**回调不跌破中枢 ZG**，此时形成的底分型为三买。

**触发条件**：
1. 存在已完成的中枢 [ZD, ZG]
2. 一笔上升突破 ZG（向上离开中枢）
3. 回调最低价 > ZG（不破中枢顶部）
4. 回调形成底分型

**图示**：
```
价格轴
  ↑              ↗（突破后再次上涨）
  |          ↗↘  ← 三买（回调不破ZG）
  |── ZG ──↗
  |      |
  |── ZD─|
  |      |（中枢震荡）
  +-------------------→ 时间
```

**操作逻辑**：
```
三买信号出现 → 果断买入（确定性最高的买点）
止损：跌破 ZG（中枢顶部）
目标：上一个更大中枢的位置
```

**核心原则**：三买是缠论最**高确定性**的买点，趋势已经确立，只是等待回调介入。

---

### 6.5 三类卖点（对称结构）

卖点与买点完全对称，方向相反：

| 类型 | 触发条件 | 操作 |
|-----|---------|-----|
| **一卖** | 上涨趋势末端，最后一笔上升笔出现背驰，顶分型形成 | 减仓/清仓 |
| **二卖** | 一卖后回调，反弹高点不破一卖高点，顶分型形成 | 清仓/做空 |
| **三卖** | 下降笔跌破中枢（< ZD），反弹不回 ZD，顶分型形成 | 坚决清仓 |

---

### 6.6 czsc 实现三买判断

```python
def check_third_buy(bi_list):
    """
    判断是否存在三买信号
    返回：('BUY3', 详情描述) 或 ('HOLD', '')
    """
    if len(bi_list) < 5:
        return ('HOLD', '笔数不足')
    
    # 识别最近中枢
    zs = find_zhongshu(bi_list[:-2])  # 排除最近2笔来找历史中枢
    if not zs:
        return ('HOLD', '未找到中枢')
    
    zg, zd = zs
    last_bi = bi_list[-1]   # 最后一笔（应为下降笔，即回调笔）
    prev_bi = bi_list[-2]   # 倒数第二笔（应为上升笔，突破中枢）
    
    # 条件1：倒数第二笔（上升笔）高点突破中枢 ZG
    if prev_bi.high <= zg:
        return ('HOLD', '未突破中枢')
    
    # 条件2：最后一笔（下降笔）低点不破中枢 ZG
    if last_bi.low < zg:
        return ('HOLD', f'回调破位 ZG={zg:.2f}, 当前低={last_bi.low:.2f}')
    
    # 条件3：最后一笔是下降笔（回调）
    from czsc.objects import Direction
    if last_bi.direction != Direction.Down:
        return ('HOLD', '最后一笔不是下降笔')
    
    detail = f"三买：ZG={zg:.2f}, 突破高={prev_bi.high:.2f}, 回调低={last_bi.low:.2f}"
    return ('BUY3', detail)
```

## 七、背驰

背驰是判断**一买/一卖**的核心依据，也是缠论最精髓的概念之一。

### 7.1 背驰的定义

**背驰**：价格创新高（或新低），但动能指标（MACD）未能同步创新高（或新低），表明趋势动能正在衰竭。

```
顶背驰（卖出信号）：
  价格：新高  ↑ > 前高
  MACD面积：  ↓ < 前次上涨MACD面积
  → 上涨动能衰竭 → 可能转折向下

底背驰（买入信号）：
  价格：新低  ↓ < 前低
  MACD面积：  ↑ > 前次下跌MACD面积（绝对值更小）
  → 下跌动能衰竭 → 可能转折向上
```

---

### 7.2 MACD 面积背驰判断方法

**计算方式**：比较两段走势中 MACD 柱子的面积（绝对值之和）

```python
import pandas as pd
import pandas_ta as ta

def calc_macd_area(closes: pd.Series, start_idx: int, end_idx: int) -> float:
    """计算指定区间的 MACD 柱面积（绝对值之和）"""
    macd_df = ta.macd(closes, fast=12, slow=26, signal=9)
    hist = macd_df['MACDh_12_26_9']  # MACD 柱
    segment = hist.iloc[start_idx:end_idx+1]
    return segment.abs().sum()

# 背驰判断示例
def check_divergence(bi_list, closes: pd.Series) -> bool:
    """
    判断最后一笔下降是否出现底背驰
    bi_list: 笔列表
    closes: 对应的收盘价序列
    """
    if len(bi_list) < 3:
        return False
    
    # 最后一笔（下降笔）
    last_bi = bi_list[-1]
    # 倒数第三笔（同向的上一段下降）
    prev_bi = bi_list[-3]
    
    # 简化：比较两段下降笔的价格幅度（用幅度替代MACD面积）
    last_amplitude = abs(last_bi.high - last_bi.low)
    prev_amplitude = abs(prev_bi.high - prev_bi.low)
    
    # 底背驰：最后一笔下降幅度 < 前一笔同向下降幅度的80%
    if last_amplitude < prev_amplitude * 0.8:
        return True  # 存在底背驰
    return False
```

### 7.3 背驰的实战意义

| 背驰类型 | 对应买卖点 | 可靠性 | 操作 |
|--------|----------|-------|-----|
| 底背驰（下降笔动能递减）| 一买 | 中等（仍可能延续）| 轻仓试探 |
| 顶背驰（上升笔动能递减）| 一卖 | 中等 | 减仓观察 |
| 中枢背驰（离开中枢后背驰）| 三卖/趋势反转 | 较高 | 重视 |

**本项目策略**：用 MACD 背驰作为辅助信号，主要依靠三买/三卖的结构确定性入场。

## 八、级别系统

### 8.1 多级别联立分析

缠论的精髓之一是**多级别联立分析**：

```
周线级别判断大方向（牛市/熊市/震荡）
    ↓
日线级别确认中期趋势
    ↓
60分钟级别寻找具体买卖点（本项目）
    ↓
30分钟级别精确入场时机（进阶）
```

**联立原则**：
- 高级别决定操作方向，不逆势操作
- 低级别寻找高确定性的入场点
- 高低级别共振时，信号最强

```

### 11.1 整体策略逻辑

```
