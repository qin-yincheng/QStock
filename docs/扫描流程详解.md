# QStock 股票池扫描完整流程详解

> 本文档基于源代码逐层剖析扫描系统的完整工作流程，包括每个判断环节的具体依据和数值标准。

---

## 一、扫描系统架构总览

```
用户发起扫描（scan.py）
    │
    ▼
StockScreenerV3.screen_pool()  ← 遍历股票池（50只/自选池/指定股票）
    │
    ├── 对每只股票执行 screen() ──┐
    │                             │
    │   ┌─────────────────────────┘
    │   │
    │   ▼
    │   Step 1: 获取 K 线数据（历史 + 实时拼接）
    │   Step 2: 计算 ADX → 低于 20 直接淘汰
    │   Step 3: 缠论笔结构判断趋势方向（up/sideways/down）
    │   Step 4: SignalEngine 融合信号（缠论 × 双龙 → 最终信号）
    │   Step 5: 计算波动率适配度（ATR/Close 比率）
    │   Step 6: 四维评分 → 总分 → 分级（推荐/关注/淘汰）
    │
    ▼
按得分排序 → 生成报告（推荐 / 关注 / 淘汰三栏）
```

## 二、扫描的 6 个步骤详解

### Step 1：获取 K 线数据

| 模式 | 数据来源 | 说明 |
|------|---------|------|
| 实时模式（默认） | Tushare 历史 + 新浪财经当天实时 K 线拼接 | 盘中使用，包含今天最新数据 |
| 纯历史模式（`--no-realtime`） | Tushare 历史数据 | 盘后使用，数据更稳定 |

- 默认获取过去 **365 天**的 K 线数据
- 默认频率 **30 分钟**（每日 8 根 K 线），可切换 F15（每日 16 根）或 F60（每日 4 根）
- 数据不足 **200 根** K 线的股票直接标记为"数据不足"并淘汰

### Step 2：ADX 趋势强度筛选（硬性门槛）

ADX（Average Directional Index）衡量趋势的强弱程度，与趋势方向无关。

**计算方式**：取最近 60 根 K 线的 ADX(14) 均值。

| ADX 范围 | 含义 | 处理 |
|---------|------|------|
| **< 20** | 弱势，无明显趋势 | **直接淘汰**，不进入后续分析 |
| 20 ~ 24 | 趋势初起 | 通过门槛，进入评分（ADX 得 10 分） |
| 25 ~ 29 | 中等趋势 | 通过门槛（ADX 得 18 分） |
| ≥ 30 | 强势趋势 | 通过门槛（ADX 得 25 分，满分） |

> **为什么 ADX < 20 直接淘汰？**
> 第七阶段 Step 7.5 的 324 组参数搜索结论：趋势跟随策略在弱趋势股上必然亏损，ADX < 20 的股票无论怎么调参都跑不出正收益。

### Step 3：缠论趋势方向判断

利用 czsc 库的 CZSC 对象识别笔结构，取**最近 3 笔**的高低点判断趋势方向：

| 条件 | 趋势 | 评分 |
|------|------|:---:|
| 最近 3 笔高点逐步抬高 **且** 低点逐步抬高 | **up（上升）** | 25 分 |
| 最近 3 笔高点逐步降低 **且** 低点逐步降低 | **down（下降）** | 0 分 |
| 其他情况（高低点方向不一致） | **sideways（震荡）** | 10 分 |

### Step 4：融合信号生成（核心步骤）

这是最复杂的一步，由 `SignalEngine` 将**缠论信号**和**双龙信号**融合为最终交易信号。

#### 4.1 缠论信号（ChanSignalGenerator）

分两层检测：

**第一层 — czsc 内置信号函数（优先）**：

| 信号函数 | 检测内容 | 可产生的信号 |
|---------|---------|------------|
| `cxt_first_buy_V221126` | 一类买点（趋势末端背驰） | BUY_1 |
| `cxt_first_sell_V221126` | 一类卖点 | SELL_1 |
| `cxt_second_bs_V230320` | 二类买卖点（回调/反弹不创新值） | BUY_2 / SELL_2 |
| `cxt_third_buy_V230228` | 三类买点（离开中枢后回踩不破） | BUY_3 |
| `cxt_five_bi_V230619` | 五笔形态（类三买/底背驰等） | BUY_5BI / SELL_5BI |
| `cxt_double_zs_V230311` | 双中枢反转 | BUY_DZS / SELL_DZS |
| `cxt_bs_V240526` | 趋势跟随买卖点 | BUY_TREND / SELL_TREND |
| `tas_macd_bc_V230803` | MACD 背驰 | BUY_DIV / SELL_DIV |

**第二层 — 笔结构手动分析（降级方案，内置信号全部无结果时启用）**：

| 检测项 | 逻辑 | 产生信号 |
|--------|------|---------|
| 三买/三卖 | 向上离开中枢后回踩不破中枢上沿 ZG | BUY_3 / SELL_3 |
| 二买/二卖 | 回调低点高于前低 / 反弹高点低于前高 | BUY_2 / SELL_2 |
| 背驰 | 同向笔幅度递减超过 10%（阈值 0.9） | BUY_DIV / SELL_DIV |

**过滤器增强**（不产生独立信号，但会调整信号强度）：

| 过滤器 | 作用 |
|--------|------|
| MACD 强弱（超强/强势/弱势/超弱） | MACD 超弱时 → 买入信号降为 HOLD；超强+多头时 → 低级别买点升级为 BUY_1 |
| MACD 方向 | 辅助判断动能方向 |
| 均线多空排列（EMA5/10/20） | 配合 MACD 强弱做升降级 |

#### 4.2 双龙信号（DragonSignalGenerator）

基于两个技术指标的金叉/死叉：

- **第一龙**：EMA5 上穿 EMA10（均线金叉）
- **第二龙**：MACD DIF 上穿 DEA（MACD 金叉）
- **共振窗口**：5 根 K 线内两龙同时出现

| 条件 | 信号 | 含义 |
|------|------|------|
| EMA 金叉 + MACD 金叉（5 根窗口内共振）+ DIF > DEA > 0 | **STRONG_BUY** | 双龙齐飞（强），零轴上方 |
| EMA 金叉 + MACD 金叉（5 根窗口内共振）+ DIF > DEA | **BUY** | 双龙齐飞，DIF 在 DEA 上方 |
| 仅 EMA 金叉 + MACD 方向向上（DIF 递增） | **BUY_EARLY** | 早期信号，仅均线转多 |
| EMA 死叉 或 MACD 死叉（任一） | **SELL** | 双龙死叉 |
| 以上都不满足 | **HOLD** | 无信号 |

**趋势状态**：
| EMA5 > EMA10 | DIF > DEA | 趋势 |
|:---:|:---:|------|
| 是 | 是 | bullish（多头） |
| 否 | 否 | bearish（空头） |
| 其他 | 其他 | neutral（中性） |

#### 4.3 信号融合规则（从上到下依次匹配，命中即停）

| 优先级 | 条件 | 最终信号 | 置信度 | 含义 |
|:---:|------|---------|:---:|------|
| 1 | 缠论出现卖出信号（SELL_1/2/3/DIV 等） | **SELL_ALL** | 85% | 缠论卖点，建议清仓 |
| 2 | 双龙死叉 且 缠论无买点 | **REDUCE** | 60% | 双龙转空，建议减仓 |
| 3 | 缠论买点 + 双龙齐飞（强） | **STRONG_BUY** | 90% | 最强共振，缠论+双龙双确认 |
| 4 | 缠论买点 + 双龙买入 | **BUY** | 80% | 缠论买点+双龙确认 |
| 5 | 缠论买点 + 双龙趋势向上（bullish） | **BUY** | 70% | 缠论买点+双龙方向支持 |
| 6 | 双龙买入 + 缠论趋势向上（up） | **BUY** | 65% | 双龙信号+缠论趋势确认 |
| 7 | 仅缠论买点，双龙未共振 | **WAIT_CONFIRM** | 55% | 缠论有信号但双龙未确认 |
| 8 | 仅双龙买入，缠论无买点 | **BUY_WATCH** | 50% | 双龙有信号但缠论未确认 |
| 9 | 双龙 EMA 金叉 + 趋势向上 | **BUY_EARLY** | 40% | 早期信号，置信度低 |
| 10 | 缠论买+双龙卖，信号矛盾 | **HOLD** | 0% | 互相矛盾，不操作 |
| 11 | 以上都不满足 | **HOLD** | 0% | 无明确信号 |

### Step 5：波动率适配度

趋势跟随策略需要适度的波动率：太低（死水）没利润，太高（疯涨疯跌）假信号多。

计算方式：最近 60 根 K 线的 **ATR / 平均收盘价**。

| ATR/Close 比率 | 波动率评价 | 得分 |
|:--------------:|----------|:---:|
| 0.8% ~ 2.5% | **最佳区间** | 15 分（满分） |
| 0.5% ~ 3.5% | 可接受 | 10 分 |
| 其他 | 过低或过高 | 5 分 |

### Step 6：四维评分与分级

**总分 = ADX 得分 + 趋势得分 + 融合信号得分 + 波动率得分**

| 维度 | 满分 | 评分标准 |
|------|:---:|---------|
| ADX 趋势强度 | **25 分** | ≥30→25, ≥25→18, ≥20→10, <20→0（已淘汰） |
| 缠论趋势方向 | **25 分** | up→25, sideways→10, down→0 |
| 融合信号强度 | **35 分** | STRONG_BUY→35, BUY→28, BUY_EARLY→18, WAIT_CONFIRM/BUY_WATCH→10, HOLD→0, SELL→-15, REDUCE→-10 |
| 波动率适配度 | **15 分** | 最佳区间→15, 可接受→10, 不适配→5 |

**分级标准**：

| 总分 | 分级 | 说明 |
|:---:|------|------|
| **≥ 55 分** | **推荐** | ADX 达标 + 趋势向上 + 有买入信号 + 波动率合适 |
| **40 ~ 54 分** | **关注** | 部分条件满足，值得持续跟踪 |
| **< 40 分** | **淘汰** | 条件不足或趋势不利 |

## 三、"推荐 / 关注 / 淘汰"具体判据拆解

### 什么样的股票会被"推荐"（≥ 55 分）？

要达到 55 分，**至少**需要满足以下组合之一：

| 路径 | ADX | 趋势 | 信号 | 波动率 | 总分 |
|------|:---:|:---:|:---:|:---:|:---:|
| 最典型的推荐 | ≥30 (25) | up (25) | BUY (28) | 最佳 (15) | **93** |
| 强信号+中等ADX | ≥25 (18) | up (25) | STRONG_BUY (35) | 可接受 (10) | **88** |
| 弱信号需要全满 | ≥30 (25) | up (25) | BUY_EARLY (18) | 最佳 (15) | **83** |
| 最低门槛组合 | ≥25 (18) | up (25) | BUY_EARLY (18) | 最佳 (15) | **76** |
| 仅 WAIT_CONFIRM | ≥30 (25) | up (25) | WAIT_CONFIRM (10) | 最佳 (15) | **75** |
| 震荡中的强信号 | ≥30 (25) | sideways (10) | STRONG_BUY (35) | 最佳 (15) | **85** |
| 刚好及格 | ≥25 (18) | sideways (10) | BUY (28) | 最佳 (15) | **71** |
| **及格最低线** | ≥20 (10) | up (25) | BUY_EARLY (18) | 可接受 (10) | **63** |

**总结**：推荐 = ADX ≥ 20（通过门槛）+ 缠论趋势向上 + 至少有 BUY_EARLY 以上的融合信号。

### 什么样的股票会被"关注"（40 ~ 54 分）？

| 典型情况 | 说明 |
|---------|------|
| ADX 达标 + 趋势向上 + 无信号（HOLD） | 趋势好但还没出信号，等待触发 |
| ADX 达标 + 震荡中 + 有弱信号 | 趋势不明朗但有信号苗头 |
| ADX 强 + 趋势下降 + 有买入信号 | 趋势不利但信号说明可能反转 |

### 什么样的股票会被"淘汰"（< 40 分）？

| 情况 | 得分 | 原因 |
|------|:---:|------|
| **ADX < 20** | 0 | 直接淘汰，不参与评分 |
| ADX 刚达标 + 下降趋势 + 无信号 | 10+0+0+5 = **15** | 弱势+空头+无信号 |
| ADX 中等 + 震荡 + 无信号 | 18+10+0+5 = **33** | 方向不明+无触发 |
| 任何组合中出现卖出信号 | 信号分 = -15 | 信号得分为负，拉低总分 |

## 四、SELL_ALL（卖出全部）的触发依据

扫描结果中 `SELL_ALL` 信号的含义是**缠论确认卖点**，它来源于融合引擎中优先级最高的规则：

> **只要缠论出现任何卖出信号 → 最终信号就是 SELL_ALL（85% 置信度）**

具体来说，以下任何一个缠论信号触发时，融合引擎都会输出 SELL_ALL：

| 缠论卖出信号 | 触发条件 | 含义 |
|-------------|---------|------|
| **SELL_1（一类卖点）** | 上涨趋势末端，czsc 内置 `cxt_first_sell_V221126` 检测到 | 趋势将要反转，最强卖出信号 |
| **SELL_2（二类卖点）** | 一卖后第一次反弹不创新高（反弹高点 < 前高） | 确认顶部结构 |
| **SELL_3（三类卖点）** | 跌破中枢后反弹不破中枢下沿 ZD | 中枢向下破位确认 |
| **SELL_5BI（五笔卖点）** | 五笔形态中出现类三卖或顶背驰 | 形态学卖出 |
| **SELL_DZS（双中枢看空）** | 双中枢结构中出现看空信号 | 结构性卖出 |
| **SELL_TREND（趋势跟随卖）** | czsc 内置 `cxt_bs_V240526` 检测到卖点 | 趋势跟随系统的卖出 |
| **SELL_DIV（MACD 顶背驰）** | 价格创新高但笔的幅度递减 > 10%，或 MACD 面积递减 | 上涨力度衰竭 |

### SELL_ALL vs REDUCE 的区别

| 信号 | 触发条件 | 操作建议 | 置信度 |
|------|---------|---------|:---:|
| **SELL_ALL** | 缠论出现卖出信号 | **清仓** | 85% |
| **REDUCE** | 双龙死叉（EMA 或 MACD 任一死叉），但缠论无卖点 | **减仓 50%** | 60% |

> 设计理念：缠论卖点代表结构性反转，严重程度高，应清仓；双龙死叉只代表短期动能减弱，可能只是回调，所以只减仓。

## 五、扫描过程中能触发止盈/止损信号吗？

**不能。** 止盈止损是回测引擎（`backtester.py`）的功能，扫描过程中不涉及。

两者的职责区分如下：

| 功能 | 扫描系统（scan.py） | 回测引擎（backtester.py） |
|------|-------------------|----------------------|
| 目的 | 发现当前有信号的股票 | 模拟历史上的持仓和交易 |
| 是否有"持仓"概念 | **否**，只看最新一根 K 线的信号 | **是**，模拟买入/持有/卖出全流程 |
| 止损 | ❌ 不涉及 | ✅ ATR 自适应止损（2.0×ATR，下限-3%~上限-8%） |
| 移动止盈 | ❌ 不涉及 | ✅ 盈利 1.5×ATR 激活，回撤 1.0×ATR 卖出 |
| 时间止损 | ❌ 不涉及 | ✅ 持仓超过 40 根 K 线仍亏损则退出 |
| 连亏暂停 | ❌ 不涉及 | ✅ 连续亏损 3 次后暂停交易 |
| T+1 约束 | ❌ 不涉及 | ✅ 买入当天不能卖出 |

**扫描能触发的卖出类信号**只有两种，且都是**基于技术指标计算的"当下状态"**，不是基于持仓盈亏：

1. **SELL_ALL**：缠论结构出现卖点（对已持仓的人是清仓信号）
2. **REDUCE**：双龙死叉（对已持仓的人是减仓信号）

如果你想看某只股票历史上止盈止损的触发情况，需要使用回测功能：
```bash
python scripts/backtest_report.py --symbol 300750.SZ --mode fused
```

## 六、完整扫描流程图（从用户输入到最终输出）

```
用户输入
  │
  ├─ "扫描自选池"          → 读取 data/portfolio.json（10只）
  ├─ "扫描市场"            → 使用默认 50 只沪深 300 成分股
  ├─ "扫描 300750.SZ"      → 指定单只或多只股票
  │
  ▼
确定参数
  ├─ 频率：F30（默认）/ F60 / F15
  ├─ 数据模式：实时+历史（默认）/ 纯历史
  ├─ ADX 门槛：20（默认）
  │
  ▼
逐只股票执行 screen() ─────────────────────────────┐
  │                                                 │
  ▼                                                 │
Step 1: 获取 K 线数据（默认 365 天，30 分钟频率）        │
  │ 数据不足 200 根？→ 淘汰（数据不足）                  │
  │                                                 │
  ▼                                                 │
Step 2: 计算 ADX（14 周期，最近 60 根均值）              │
  │ ADX < 20？→ 淘汰（弱势股）                         │
  │                                                 │
  ▼                                                 │
Step 3: 缠论趋势判断                                   │
  │ 创建 CZSC 对象 → 识别笔结构                        │
  │ 取最近 3 笔 → 判断 up / sideways / down            │
  │                                                 │
  ▼                                                 │
Step 4: 融合信号计算                                   │
  │ ┌─ ChanSignalGenerator ─────────────────┐        │
  │ │ 8 个 czsc 内置信号函数逐一检测          │        │
  │ │ 全部为"其他"时退化为笔结构手动分析      │        │
  │ │ 3 个过滤器做信号升降级                  │        │
  │ └─ 输出：缠论信号 + 趋势 ──────────────┘        │
  │                                                 │
  │ ┌─ DragonSignalGenerator ───────────────┐        │
  │ │ 计算 EMA5/10、DIF/DEA/MACD 柱         │        │
  │ │ 判断金叉/死叉 + 5 根窗口内共振          │        │
  │ │ 输出趋势状态 bullish/bearish/neutral    │        │
  │ └─ 输出：双龙信号 + 趋势 ──────────────┘        │
  │                                                 │
  │ ┌─ 融合规则（11 条，从上到下匹配）─────────┐        │
  │ │ 缠论卖 → SELL_ALL                      │        │
  │ │ 双龙卖+缠论无买 → REDUCE                │        │
  │ │ 缠论买+双龙强买 → STRONG_BUY            │        │
  │ │ ... 共 11 条规则 ...                    │        │
  │ │ 都不满足 → HOLD                         │        │
  │ └─ 输出：最终信号 + 置信度 ────────────┘        │
  │                                                 │
  ▼                                                 │
Step 5: 波动率适配度                                   │
  │ ATR(60根) / 平均收盘价                             │
  │ 0.8%~2.5% → 15分, 0.5%~3.5% → 10分, 其他 → 5分   │
  │                                                 │
  ▼                                                 │
Step 6: 综合评分                                       │
  │ 总分 = ADX分 + 趋势分 + 信号分 + 波动率分           │
  │ ≥55 → 推荐, 40~54 → 关注, <40 → 淘汰             │
  │                                                 │
  ▼                                                 │
仓位映射（仅推荐/关注的买入信号）                        │
  │ STRONG_BUY → 40%                                │
  │ BUY → 30%                                       │
```

## 八、扫描系统的局限性

1. **扫描只看"最新一根 K 线"的快照信号**，不追踪持仓状态，无法判断止盈止损
2. **信号有时效性**：30 分钟频率下，信号每 30 分钟更新一次，盘中需要在 K 线收盘后立即扫描
3. **ADX 是滞后指标**：ADX 上升说明趋势正在变强，但趋势初期 ADX 仍可能 < 20
4. **缠论信号依赖数据量**：笔数量不足 3 笔时无法判断趋势，不足 5 笔时手动分析不可用
5. **50 只股票全量扫描耗时 2-5 分钟**，受 Tushare 限频影响
