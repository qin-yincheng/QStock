## 三、判断是否需要更新技能文件

### 快速判断规则

| 你修改了什么 | 需要更新技能吗 | 说明 |
|-------------|:---:|------|
| `scripts/scan.py` | 否 | 技能只是调用脚本，脚本更新后自动生效 |
| `scripts/portfolio.py` 新增参数 | **是** | 需要更新 `qstock-portfolio` 和/或 `qstock-holding` 的 SKILL.md |
| `scripts/backtest_report.py` 新增参数 | **是** | 需要更新 `qstock-backtest/SKILL.md` |
| `scripts/analyze.py` 新增参数 | **是** | 需要更新 `qstock-advisor/SKILL.md` |
| `tools/stock_screener_v3.py` | 否 | 被 scan.py 内部调用，不影响技能接口 |
| `strategy/*.py` | 否 | 信号引擎内部逻辑，不影响技能接口 |
| `backtest/backtester.py` | 否 | 被 backtest_report.py 内部调用 |
| `data/portfolio.json` 结构变更 | **是** | 需要更新 `qstock-portfolio` 和 `qstock-holding` 的 SKILL.md |
| 新增脚本 | **是** | 需要创建新的 SKILL.md 或更新已有技能 |
| 只改了 `docs/` 下的文档 | 否 | 文档不影响技能运行 |

**核心原则**：技能文件（SKILL.md）只是告诉龙虾"怎么调用脚本"。如果脚本的**命令行接口不变**（参数名、参数含义没变），就不需要更新技能文件。

### 当前已注册的 5 个技能

| 技能 | 技能路径 | 对应脚本 | 关键参数 |
|------|---------|---------|---------|
| qstock-scanner | `skills/qstock-scanner/SKILL.md` | `scripts/scan.py` | `--portfolio` `--symbols` `--freq` `--output` |
| qstock-backtest | `skills/qstock-backtest/SKILL.md` | `scripts/backtest_report.py` | `--symbol` `--mode` `--freq` `--sdt` `--edt` |
| qstock-portfolio | `skills/qstock-portfolio/SKILL.md` | `scripts/portfolio.py` | `--action list/add/remove/diagnose/backtest-all` |
| qstock-advisor | `skills/qstock-advisor/SKILL.md` | `scripts/analyze.py` | `--symbol` `--freq` |
| qstock-holding | `skills/qstock-holding/SKILL.md` | `scripts/portfolio.py` | `--action update-holding/clear-holding/holdings` |

```

          │

```

## 六、常见问题

### Q1：git pull 报冲突怎么办？

```
QStock 代码更新时 git pull 报冲突了。请执行以下命令强制同步到远程最新版本：

cd /home/admin/openclaw/workspace/QStock
git fetch origin
git reset --hard origin/main

注意：这会丢弃龙虾平台上的本地修改，以 GitHub 上的版本为准。
```

### Q2：pip 依赖有更新怎么办？

```
QStock 依赖有更新，请重新安装：
cd /home/admin/openclaw/workspace/QStock
source venv/bin/activate
pip install -r requirements.txt
```

### Q3：怎么确认龙虾用的是最新代码？

```
请帮我检查 QStock 代码版本：
cd /home/admin/openclaw/workspace/QStock
git log --oneline -3
```

### Q4：portfolio.json 在龙虾上被修改了，git pull 会冲突吗？

不会。`data/portfolio.json` 已在 `.gitignore` 中，龙虾平台上的持仓数据独立于 GitHub，互不影响。

> **注意**：如果之前已经提交过 `portfolio.json` 到 git 仓库，需要在龙虾上执行一次取消跟踪：
> ```
> cd /home/admin/openclaw/workspace/QStock
> git rm --cached data/portfolio.json
> ```
> 之后 git pull 就不会再冲突了。
