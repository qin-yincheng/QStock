# 扫描系统问题分析与改进方案

> 基于龙虾实际扫描结果（`docs/龙虾.md`）暴露的问题，结合源代码深度分析。

---

## 问题一：扫描结果缺乏真正的缘由，且几乎所有股票都有信号

### 1.1 现象描述

从龙虾的实际扫描结果来看：

```
10 只股票中：
  - 1 只推荐（比亚迪 BUY_1）
  - 3 只关注（宁德时代 SELL_ALL、五粮液 HOLD、中国平安 HOLD）
  - 6 只淘汰（其中 5 只是 SELL_ALL）
```

**问题 1**：10 只股票中有 **6 只都是 SELL_ALL**（卖出全部），看起来"到处都有信号"。

**问题 2**：扫描只给出信号名称（如 `BUY_1`、`SELL_3`），但没有说明**具体的触发价位和结构细节**——比如"中枢在哪""回踩到了什么位置""背驰的两笔幅度分别是多少"。

**问题 3**：没有区分**已持仓**和**未持仓**该怎么做。

### 1.2 根因分析

#### 为什么大量出现 SELL_ALL？

翻看源代码中的融合规则（`signal_engine.py` 第 129 行）：

```python
if chan_sell:
    return {"action": "SELL_ALL", "confidence": 0.85, ...}
```

**只要缠论出现任何卖出信号，就直接输出 SELL_ALL。** 而缠论的卖出信号有 7 种之多（SELL_1/2/3/5BI/DZS/TREND/DIV），覆盖面非常广。在任何一个时间点，大量股票处于某种"结构性卖点"是正常的——因为缠论的卖点不代表"现在必须卖"，而是"结构上走到了一个可能反转的位置"。

**核心问题**：扫描系统把缠论的"结构性判断"直接翻译成了"操作指令"，缺少了中间的**过滤和解释环节**。

#### 为什么缺乏具体缘由？

扫描流程中，`ChanSignalGenerator.analyze()` 确实会返回一个 `detail` 字段，例如：

```
"三类卖点: 反弹不破中枢下沿 ZD=185.50, 高点=183.20"
```

但这个信息在 `StockScreenerV3.screen()` 中**没有被保存到结果**里——它只保留了 `signal`（信号名称）和 `reason`（评分构成），丢失了缠论的结构细节。

#### 为什么没有持仓/未持仓区分？

扫描系统的 `screen()` 函数只分析"最新一根 K 线的快照状态"，完全没有"当前是否持仓"的概念。它不知道：
- 你是否已经买入了这只股票
- 如果买入了，买入价是多少
- 当前浮盈/浮亏多少
- 应该继续持有、止盈、还是止损

### 1.3 举例说明

以龙虾结果中的**宁德时代**为例：

**龙虾给出的信息**：
```
宁德时代 300750.SZ  卖出(全)  ADX=43.4  趋势=上升  得分=50
```

**缺少的关键信息**：
- 卖出信号具体是什么类型？（SELL_1 一卖？SELL_3 三卖？SELL_DIV 背驰？）
- 中枢区间是多少？（ZG=? ZD=?）
- 触发卖出的具体笔结构？（最近一笔从哪到哪，背驰幅度多少？）
- **如果我已持仓**：买入价是多少？当前浮盈还是浮亏？应该立即卖还是等回弹？
- **如果我未持仓**：是否应该做空（A 股不能）？还是等跌完后接回来？

## 问题二：缺少止盈止损策略，实战会不会亏更多？

### 2.1 回测 vs 实战的关键差异

这是整个系统**最致命的问题**。回测引擎有 **5 层保护机制**，但扫描系统一层都没有：

| 保护机制 | 回测引擎（backtester.py） | 扫描系统（scan.py） |
|---------|:---:|:---:|
| **ATR 自适应止损**（亏损达 2×ATR 自动卖出） | ✅ | ❌ |
| **移动止盈**（盈利 1.5×ATR 后回撤 1×ATR 锁利） | ✅ | ❌ |
| **时间止损**（持仓 40 根 K 线仍亏损则退出） | ✅ | ❌ |
| **连亏暂停**（连续亏损 3 次后暂停交易） | ✅ | ❌ |
| **量能过滤**（缩量时缩小仓位） | ✅ | ❌ |

### 2.2 没有止损意味着什么？

用具体数字说明。假设回测结果显示宁德时代过去一年收益 +8.33%，以下是有止损和无止损的对比：

**有止损（回测环境）**：
```
交易 1: 买入 190 → 跌到 182 → ATR 止损触发 → 卖出 182 → 亏损 -4.2%
交易 2: 买入 185 → 涨到 210 → 移动止盈触发（210 回撤到 205）→ 卖出 205 → 盈利 +10.8%
交易 3: 买入 195 → 持仓 40 根 K 线 → 时间止损 → 卖出 193 → 亏损 -1.0%
净结果: -4.2% + 10.8% - 1.0% ≈ +5.6%
```

**无止损（纯信号操作）**：
```
交易 1: 买入 190 → 跌到 182 → 没有止损，继续持有 → 跌到 170 → 终于出信号 SELL_ALL → 亏损 -10.5%
交易 2: 买入 185 → 涨到 210 → 没有止盈，继续持有 → 回落到 188 → 终于出信号 SELL_ALL → 盈利 +1.6%
交易 3: 买入 195 → 横盘震荡 → 没有时间止损 → 一直耗着 → 资金被占用无法参与其他机会
净结果: -10.5% + 1.6% + 0% = -8.9%
```

**关键差异**：
- 止损把单笔最大亏损限制在 **-3% ~ -8%**，无止损可能亏 **-10% 以上**
- 移动止盈在回撤初期就锁住利润，无止盈则坐过山车
- 时间止损释放被横盘占用的资金

### 2.3 结论：没有止盈止损，实战一定比回测亏更多

**是的，没有止盈止损的实战操作，亏损一定会比回测结果更大。** 原因：

1. **回测中的正收益很大程度上是止盈止损贡献的**——它们不只是防守工具，更是利润来源
2. 回测的 v8 策略经过 8 个阶段优化，ATR 止损+量能过滤贡献了盈亏比从 1.31 提升到 1.75（+34%），回撤从 -1.83% 降到 -1.52%（-17%）
3. 如果只用信号不用止损，等于**退回到了 v1 版本之前的状态**，连第四阶段的基本回测都跑不过

## 问题的根源：扫描系统和回测引擎是两套独立的逻辑

```
                    ┌──────────────────┐
                    │    回测引擎       │
                    │  backtester.py   │
                    │                  │
                    │  ✅ 信号生成      │
                    │  ✅ 买入执行      │
                    │  ✅ 持仓追踪      │
                    │  ✅ ATR 止损      │
                    │  ✅ 移动止盈      │
                    │  ✅ 时间止损      │
                    │  ✅ 连亏暂停      │
                    │  ✅ 量能过滤      │
                    │  ✅ 分层退出      │
                    └──────────────────┘
                           ↑
                     历史模拟（完整闭环）
                           │
                    ┌──────────────────┐
                    │  但现在的扫描是    │
                    │  完全独立的另一条路 │
                    └──────────────────┘
                           │
                    ┌──────────────────┐
                    │    扫描系统       │
                    │    scan.py       │
                    │                  │
                    │  ✅ 信号生成      │
                    │  ❌ 没有买入执行   │
                    │  ❌ 没有持仓追踪   │
                    │  ❌ 没有止损       │
                    │  ❌ 没有止盈       │
                    │  ❌ 没有时间止损   │
                    │  ❌ 没有连亏暂停   │
                    │  ❌ 量能只参与评分  │
                    │  ❌ 没有退出管理   │
                    └──────────────────┘
```

**扫描系统只做了回测引擎的第一步（信号生成），后面的 8 步全部缺失。**

## 改进方案

### 方案 A：在扫描结果中附带具体操作参数（推荐，改动最小）

**思路**：扫描时不仅输出信号，还计算并输出"如果按此信号操作，止损/止盈/仓位具体是多少"。

**具体改动**：

当扫描输出买入信号时，额外计算并展示：

```
比亚迪 (002594.SZ) — 买入信号 (BUY_1)

  缠论结构：一类买点，下跌趋势末端背驰
    - 最近中枢区间：[285.0, 298.5]
    - 当前价位 302.8 已离开中枢上方
    - 背驰依据：最后下跌笔幅度 12.3 < 前一下跌笔 18.7 (比值 0.66 < 0.9)

  双龙状态：EMA5=301.2 > EMA10=298.5 (均线多头)，DIF > DEA (MACD 多头)

  ┌─────── 操作参数（基于 v8 策略）──────┐
  │                                       │
  │  建议买入价：302.8（当前价）            │
  │  建议仓位：30%（BUY 信号对应仓位）      │
  │                                       │
  │  ATR(14) = 5.83                       │
  │  止损价：302.8 - 2×5.83 = 291.1       │
  │  止损幅度：-3.9%（在 -3%~-8% 范围内）  │
  │                                       │
  │  止盈激活：302.8 + 1.5×5.83 = 311.5   │
  │  止盈回撤：达到最高价后回落 1×ATR 卖出  │
  │                                       │
  │  时间止损：买入后 40 根 K 线（约 5 天）  │
  │           仍亏损则退出                  │
  │                                       │
  │  连亏暂停：如果已连亏 3 次，本次不操作   │
  └───────────────────────────────────────┘

  已持仓：继续持有，止损价设在 291.1
  未持仓：可在 302.8 附近建仓 30%，严格设好止损
```

当扫描输出卖出信号时：

```
长江电力 (600900.SH) — 卖出全部信号 (SELL_ALL)

  缠论结构：二类卖点 (SELL_2)
    - 上涨后第一次回调反弹不创新高
    - 反弹高点 28.35 < 前高 29.12
    - 意味着上涨动力衰竭

  双龙状态：EMA5 < EMA10 (均线空头)

  已持仓：建议清仓。如果浮亏超过 -5%，说明已经错过了最佳卖点，
          当前缠论确认卖点是最后的退出机会。
  未持仓：不操作，等待后续下跌企稳后的买入机会。

  关注价位：如果回调到 26.50（前低附近）企稳不破，可能形成二买。
```

### 方案 B：构建"持仓监控"模块（中等改动）

**思路**：新增一个持仓追踪模块，用户买入后记录到 portfolio.json 中（含买入价、买入时间），扫描时根据持仓状态给出不同建议。

**持仓数据扩展**：

```json
{
  "stocks": [
    {
      "symbol": "300750.SZ",
      "name": "宁德时代",
      "added_date": "2026-03-20",
      "holding": true,
      "buy_price": 195.50,
      "buy_date": "2026-03-21",
      "buy_bars": 0,
      "shares": 500,
      "atr_at_buy": 5.83,
      "stop_loss_price": 183.84,
      "trailing_activate_price": 204.25
    }
  ]
}
```

**扫描时的差异化建议**：

| 状态 | 买入信号 | 卖出信号 | HOLD |
|------|---------|---------|------|
| **未持仓** | 给出买入价+止损价+仓位 | 不操作，标记"等待买点" | 不操作 |
| **已持仓且浮盈** | 可加仓（如信号更强） | 执行卖出，计算实际盈亏 | 持有，检查移动止盈 |
| **已持仓且浮亏** | 不加仓（已亏不追） | 执行止损 | 持有，检查止损线 |
| **已持仓且超时** | 不加仓 | 执行卖出 | 触发时间止损 |

### 方案 C：实时交易引擎（大改动，长期目标）

**思路**：将回测引擎改造为"实时模式"，每根 K 线更新时推进一步，维护真实的持仓状态。

这本质上是把 `backtester.py` 的 `_simulate()` 改为**逐步执行**：
- 每次 K 线收盘后，执行一步模拟
- 读取当前持仓状态
- 检查止损/止盈/信号
- 输出操作建议
- 更新持仓状态

这个方案可以完全复用回测引擎的所有逻辑，但需要持久化存储持仓状态。

## 对龙虾扫描结果的具体诊断

回到 `docs/龙虾.md` 中的实际结果，逐一分析问题：

### 问题 1："6 只 SELL_ALL"是否合理？

**部分合理，但展示方式有误导性。**

从 60 分钟频率的回测对比（龙虾.md 第二部分）可以看到：
- 中国平安、五粮液、东方财富、药明康德在 60 分钟上都是 SELL_3（三类卖点）
- 这说明这些股票确实走到了"跌破中枢后反弹不破中枢下沿"的结构

但问题是：
- **SELL_ALL 信号对未持仓的人毫无意义**——你没有仓位可以"全部卖出"
- **SELL_ALL 信号对已持仓但已设好止损的人意义有限**——止损已经在保护你了
- 扫描应该区分：这是"给持仓者的清仓建议"还是"给观望者的回避提示"

### 问题 2：龙虾为什么不说明具体缘由？

因为 `StockScreenerV3.screen()` 返回的 `result` 字典中：
- `signal` = 信号名称（如 `SELL_ALL`）
- `chan_signal` = 缠论原始信号（如 `SELL_3`）
- `reason` = 评分构成（如 `ADX=43.4(25分) | 趋势=up(25分) | 信号=SELL_ALL(-15分) | 波动率=1.08%(15分)`）

但**缠论信号的具体结构细节**（中枢位置、笔的幅度、背驰比值等）保存在 `ChanSignalGenerator.analyze()` 的 `detail` 字段中，这个字段**没有传递到 screener 的结果里**。

龙虾看到的只是 `SELL_ALL` 和评分构成，所以它只能说"缠论出现卖出信号"，无法说"三类卖点：反弹高点 183.20 未能突破中枢下沿 185.50"。

### 问题 3："30 分钟 vs 60 分钟信号冲突"的处理

龙虾的分析结果显示 4 只股票都是"30 分钟持仓 vs 60 分钟卖出"。

龙虾给出了"优先看大级别"的建议，这在缠论框架下是正确的。但更准确的说法应该是：

> 60 分钟出现卖点意味着**中期结构转弱**，30 分钟的"持仓等待"只代表短期没有进一步恶化。
> 大级别方向（60 分钟向下）决定了中期趋势，小级别（30 分钟震荡）只是在大级别下跌中的一个喘息。
> 
> - **已持仓**：以 60 分钟卖点为主，减仓或清仓
> - **未持仓**：不要因为 30 分钟"持仓等待"就入场，等 60 分钟出现买点再说

## 总结

| 问题 | 根因 | 影响 |
|------|------|------|
| 信号缺乏具体缘由 | screener 丢弃了缠论的 `detail` 字段 | 用户无法判断信号可信度 |
| 几乎所有股票都有信号 | 缠论卖出信号覆盖面广，没有按重要性分级 | 信号泛滥导致信息过载 |
| 不区分已持仓/未持仓 | 扫描系统没有持仓状态概念 | 同一个 SELL_ALL 对持仓者和观望者含义完全不同 |
| 没有止损止盈参数 | 扫描系统和回测引擎完全独立 | **实战操作必然比回测亏更多** |
| 回测表现无法复现 | 回测的 5 层保护在实战中缺失 | 回测 +8.33% 的结果在实战中可能变成 -5% |

**最关键的一句话**：回测引擎的收益有一大半是止盈止损机制贡献的，如果实战中不执行同样的止盈止损，回测结果就是"虚假承诺"。
