## 二、技术背景知识

### 2.1 缠论（缠中说禅技术分析理论）

缠论是"缠中说禅"（李彪，笔名）在其博客中提出的一套完整的股票技术分析理论，共108课。核心概念自底向上构建：

#### 四层基本构件

```
K线 → 分型 → 笔 → 线段 → 中枢 → 走势类型 → 买卖点
```

**（1）分型**
- **顶分型**：三根K线中，中间一根的高点最高、低点也最高（像山顶）
- **底分型**：三根K线中，中间一根的低点最低、高点也最低（像谷底）
- 这是缠论的最小识别单元

**（2）笔**
- 至少由5根K线构成
- **上升笔**：底分型的最低点 → 顶分型的最高点
- **下降笔**：顶分型的最高点 → 底分型的最低点
- 相邻笔之间必须方向交替

**（3）线段**
- 至少由3笔组成
- 同方向的连续推进构成一个线段
- 线段的破坏需要反方向出现至少3笔

**（4）中枢**
- 至少3笔重叠区域构成
- 中枢有明确的上沿（ZG）和下沿（ZD）
- 中枢是判断趋势和震荡的核心

#### 三类买卖点

| 类型 | 买点条件 | 卖点条件 | 意义 |
|------|---------|---------|------|
| **第一类** | 下跌趋势最后一个中枢的背驰点 | 上涨趋势最后一个中枢的背驰点 | 趋势反转信号 |
| **第二类** | 第一类买点后的第一次回调结束点 | 第一类卖点后的第一次反弹结束点 | 确认趋势反转 |
| **第三类** | 次级别离开中枢后回试不破中枢上沿 | 次级别离开中枢后回抽不破中枢下沿 | 中枢方向确认 |

#### 量化实现难点

- **分型包含处理**：K线存在包含关系时需要合并处理，规则复杂
- **笔的划分争议**：新笔/旧笔划分方式不同流派有不同理解
- **中枢递归**：不同级别的中枢会嵌套递归，实现复杂度高
- **背驰判断**：需要对比MACD面积、力度等，有一定主观性

### 2.2 双龙战法

双龙战法是一种组合技术指标的短线交易策略。"双龙"指两个技术信号的共振确认：

#### 常见定义（MACD + 均线组合）

**第一龙 — 均线金叉**：
- 短期均线（如5日/MA5）上穿长期均线（如10日/MA10或20日/MA20）
- 表明短期趋势向上转强

**第二龙 — MACD确认**：
- MACD的DIF线上穿DEA线形成金叉
- 且MACD柱线由负转正（或红柱放大）
- 表明中期动能转多

**核心逻辑**：两龙齐飞（均线金叉 + MACD金叉同时出现）= 强买入信号

#### 买卖规则

| 操作 | 条件 |
|------|------|
| 买入 | MA5上穿MA10 **且** MACD金叉/红柱放大 |
| 卖出 | MA5下穿MA10 **或** MACD死叉/绿柱放大 |
| 止损 | 跌破近期支撑位（前低）或MA10 |
| 持仓周期 | 通常3-5个交易日（短线） |

#### 量化实现相对简单

双龙战法的量化实现远比缠论简单，因为它只涉及均线和MACD两个标准技术指标的计算，所有量化库（TA-Lib、pandas_ta）都直接支持。

## 四、可行方案分析

### 方案一：AI视觉分析路线（原文方案复刻）

**原理**：让 OpenClaw Agent 截取K线图，用多模态大模型识别缠论形态和双龙信号。

```
定时触发 → Agent打开财经网站 → 截图不同周期K线 → 多模态AI分析 → 输出买卖建议
```

**优点**：
- 实现最简单，几乎不需要写策略代码
- 直接复刻原文方案，有现成参考
- 利用大模型的图像理解能力，可以识别复杂的缠论形态
- 对缠论理解不深也能快速上手

**缺点**：
- 🔴 **精度存疑**：大模型的图像分析不如数值计算精确，容易误判
- 🔴 **不可回测**：无法对历史数据进行系统性回测验证
- 🟡 **延迟较高**：截图→上传→AI分析→返回结果，可能需要10-30秒
- 🟡 **成本问题**：多模态模型调用消耗token较多，免费额度有限
- 🟡 **不确定性**：同一张图多次分析可能给出不同结论

**适合场景**：快速出 Demo、验证概念可行性

### 方案二：传统量化路线（代码计算策略）

**原理**：以 czsc 生态为核心，用其内置 Tushare 连接器获取数据、CZSC 对象做缠论分析，配合自定义双龙信号，Agent 负责调度和通知。

```
Tushare (czsc.ts_connector) → czsc CZSC对象(缠论) → pandas_ta(双龙) → 信号融合 → Agent推送
```

**优点**：
- ✅ **精确可控**：数值计算结果确定，无随机性
- ✅ **可回测**：可以对历史数据系统验证策略效果
- ✅ **低延迟**：数据计算在毫秒级完成
- ✅ **低成本**：不需要频繁调用大模型
- ✅ **可优化**：参数可以调优，策略可以迭代

**缺点**：
- 🟡 缠论算法实现复杂度较高（好在有现成开源库 czsc）
- 🟡 需要一定的量化编程基础
- 🟡 需要理解缠论概念才能正确使用和调试

**适合场景**：严肃的策略验证、长期可持续的量化系统

### 方案三：混合路线（推荐）

**原理**：以 czsc 生态为核心做数据采集和策略计算，用 AI 做辅助分析和异常情况判断。

```
阶段1: Tushare(czsc.ts_connector) → CZSC对象(缠论) → pandas_ta(双龙) → 信号融合
阶段2: 信号触发时截图K线 → 多模态AI(缠论思维,参考czsc_skills)做二次确认
阶段3: Agent 整合结果 → 推送通知（含图表+分析文字）
```

**优点**：
- 兼具精确计算和 AI 理解力
- 可回测的同时又有 AI 辅助判断
- 信号经过双重过滤，质量更高
- 充分发挥 OpenClaw Agent 的能力

**缺点**：
- 开发工作量最大
- 系统复杂度较高

**双龙战法核心逻辑**：

```python
import pandas as pd
import talib

def double_dragon_signal(df):
    """双龙战法信号生成"""
    df['ma5'] = df['close'].rolling(5).mean()
    df['ma10'] = df['close'].rolling(10).mean()
    
    macd, signal, hist = talib.MACD(
        df['close'].values, 
        fastperiod=12, slowperiod=26, signalperiod=9
    )
    
    # 第一龙：MA5 上穿 MA10
    ma_cross_up = (df['ma5'] > df['ma10']) & (df['ma5'].shift(1) <= df['ma10'].shift(1))
    
    # 第二龙：MACD 柱由负转正
    macd_turn_pos = (hist > 0) & (pd.Series(hist).shift(1) <= 0)
    
    # 双龙齐飞（两个信号在3根K线内先后出现即视为共振）
    buy_signal = ma_cross_up & macd_turn_pos
    
    return buy_signal
```

**信号融合思路**：

```python
def combined_signal(chan_signal, dragon_signal):
    """
    缠论提供方向判断（趋势/震荡/买卖点级别）
    双龙提供入场时机（短线共振确认）
    
    策略：缠论出现买点（一买/二买/三买） + 双龙确认 → 强买入
    """
    if chan_signal == 'buy_point' and dragon_signal == 'buy':
        return 'STRONG_BUY'
    elif chan_signal == 'buy_point':
        return 'WEAK_BUY'
    elif dragon_signal == 'sell':
        return 'SELL'
    return 'HOLD'
```

## 六、核心技术难点与应对

### 6.1 实时数据（导师特别提到的难点）

**问题**：60分钟K线的"实时"需求意味着什么？

实际上，60分钟级别策略并不需要毫秒级实时数据。每60分钟产生一根新K线，我们只需要在每根K线收盘时获取最新数据并运行策略即可。

**应对方案**：

| 级别 | 数据更新频率 | 延迟容忍度 | 推荐方案 |
|------|------------|-----------|---------|
| 60分钟 | 每小时1次 | 分钟级 | akshare 定时拉取（REST API），足够使用 |
| 30分钟 | 每30分钟1次 | 30秒级 | akshare 定时拉取，基本满足 |
| 15分钟 | 每15分钟1次 | 10秒级 | 可能需要 WebSocket 实时推送 |

**关键结论**：对于60分钟级别，akshare 的 REST API 定时轮询完全满足需求，不存在真正的"实时数据"难题。只有扩展到15分钟及以下级别时，才需要考虑 WebSocket 方案。

#### ⚠️ 回测与实战的数据拼接问题

回测时用 baostock 获取完整历史数据即可。但**进入实战阶段**时存在一个关键问题：

- baostock 的当天数据要到**晚上20:30**才入库，盘中无法使用
- akshare 只能获取**最近5个交易日**（约20根60分钟K线）的数据
- 缠论分析（czsc）需要**较长的K线序列**（数百根）才能正确识别分型→笔→中枢

**实战数据方案 — 历史+实时拼接**：

```
baostock历史数据(截至昨天) + akshare最新数据(包含今天盘中) → 去重拼接 → 喂给策略引擎
```

具体实现：每天盘前用 baostock 拉取截至昨日的全部历史，缓存到本地；盘中定时用 akshare 拉取最近5天数据，与历史数据拼接（按时间去重），形成完整序列后再运行策略。详见可执行步骤文档 Step 1.5 中 `DataFetcher.get_merged_60min()` 方法。

**定时拉取方案代码框架**：

```python
import schedule
import time

def run_strategy():
    """每小时运行一次策略"""
    # 1. 拉取最新60分钟K线
    df = ak.stock_zh_a_hist_min_em(symbol="600519", period="60", ...)
    
    # 2. 计算缠论形态
    c = CZSC(convert_to_bars(df))
    chan_signal = analyze_chan(c)
    
    # 3. 计算双龙信号
    dragon_signal = double_dragon_signal(df)
    
    # 4. 综合判断
    final = combined_signal(chan_signal, dragon_signal)
    
    # 5. 推送通知
    if final in ['STRONG_BUY', 'SELL']:
        notify_via_agent(final)

# A股交易时段：9:30-11:30, 13:00-15:00
# 60分钟K线收盘时间：10:30, 11:30, 14:00, 15:00
schedule.every().day.at("10:31").do(run_strategy)
schedule.every().day.at("11:31").do(run_strategy)
schedule.every().day.at("14:01").do(run_strategy)
schedule.every().day.at("15:01").do(run_strategy)
```

### 6.2 缠论算法实现复杂度

**问题**：缠论完整实现涉及分型包含处理、笔段递归、中枢生长等复杂逻辑。

**应对**：直接使用 **czsc** 开源库（https://github.com/waditu/czsc）

- 4700+ Stars，持续活跃维护
- 覆盖分型、笔、线段、中枢的完整算法
- 内置信号函数和事件驱动框架
- 有配套的 Bilibili 视频教程和文档
- 支持 `pip install czsc` 一键安装

**风险**：czsc 库的学习曲线仍然不低，需要先理解缠论基本概念才能正确使用。建议投入 3-5 天专门学习缠论理论。

### 6.3 A 股 T+1 交易限制

**问题**：A 股实行 T+1 交易制度，当天买入的股票次日才能卖出。这对短周期策略影响很大。

**应对**：
- 60 分钟级别策略天然适应 T+1（持仓通常跨天）
- 30/15 分钟级别策略需要特别注意：信号出现后不能当天反转
- 回测时必须加入 T+1 约束，否则结果失真
- 可考虑 ETF（如创业板ETF 159915）作为标的，部分 ETF 支持 T+0

#### 两种T+1回测模式

| 模式 | 假设 | 适用场景 | T+0可能性 |
|------|------|---------|----------|
| **纯T+1模式** | 空仓进场，买入后至少次日才能卖出 | 系统扫描新机会，无预设持仓 | 不可T+0 |
| **持仓T+0模式** | 假设已持有该股票一定仓位 | 对自选股做日内波段操作 | 可做变相T+0（先买后卖原有持仓） |

当前系统默认采用**纯T+1模式**（适合扫描发现新机会）。如后续导师需要对已持仓股票做日内分析，可扩展为持仓T+0模式。

### 6.4 股票池来源与扫描策略

**问题**：系统应该分析哪些股票？

**需与导师确认的选项**：

| 选项 | 说明 | 工作量 |
|------|------|--------|
| A. 导师指定股票列表 | 导师提供他关注/持仓的股票，系统对这些股票做技术分析 | 最小 |
| B. 固定股票池扫描 | 如沪深300成分股、行业龙头池等，系统定时全量扫描 | 中等 |
| C. 智能选股+分析 | 系统先根据量价特征筛选强势股，再做缠论+双龙分析 | 最大 |

**建议**：初期采用 **A+B 结合** — 导师提供核心关注列表（10-20只），再加上一个热门板块龙头池作为补充扫描范围。这样既满足导师的个性化需求，又有一定的市场覆盖面。

### 6.5 信号质量与过拟合

**问题**：技术分析策略容易在历史数据上过拟合。

**应对**：
- 样本内/样本外分离：用 2023-2025 年数据训练，2026 年数据验证
- 参数不宜过多：双龙战法参数尽量用默认值（MA5/10, MACD 12/26/9）
- 多标的验证：不只测一只股票，至少测沪深300成分股中的多只
- 交易成本建模：回测中加入手续费（万三）和滑点（1-2个tick）

## 十、风险与建议

### 核心风险

1. **缠论学习曲线**：缠论理论体系庞大（108课），完全掌握需要大量时间。建议只学习核心概念（分型→笔→中枢→三类买卖点），借助 czsc 库降低实现难度。

2. **免费额度耗尽**：JVS Claw 的免费额度有限。建议在本地开发调试策略代码，只在需要 Agent 集成时才使用云平台。

3. **数据质量问题**：akshare 依赖爬虫，可能有反爬限制或数据缺失。建议做好异常处理和数据校验。

4. **策略有效性**：纯技术面策略在A股的有效性一直存在争议。保持合理预期，重点在于系统构建能力的锻炼，而非追求高收益。

### 给导师的建议回复要点

1. **可行性**：完全可行。原文方案已有成功实践，且有成熟开源工具支持。
2. **实时数据**：60分钟级别不存在实时数据难题，akshare 定时拉取即可满足。
3. **技术路线**：建议混合路线 — 代码计算为主（精确可回测），AI 视觉分析为辅（辅助判断）。
4. **时间估计**：从零到可演示的系统，约需6-8周。
5. **需要确认**：监控的股票范围（全市场扫描 vs 指定股票池）、是否需要模拟交易功能。
